# Design — Syllabus generation + control-number serials (`reg-syllabus`)

- **Date:** 2026-06-09
- **Status:** Approved (MVP cut: stamp + build + register)
- **Module:** new `lectern.syllabus` + `lectern.syllabus_serial`
- **Doctrine (the design source):** [[notes/syllabus-doctrine]] · register [[notes/syllabus-serial-register]] · framework [[notes/serialized-artifact-doctrine]]
- **First user:** `~/git/cecs-378-su26-01-syllabus-10660` (live Su26 syllabus repo)

## Problem

Syllabi are generated by ad-hoc per-repo scripts (`gen_output.sh`, `gen_canvas.py`,
`eisvogel.latex`, `syllabus.css` copied into each repo) and the control-number
**serial** is computed/injected by hand (the Su26 repo already carries
`serial: F66CDFA2` + `<footer class="serial">Serial · F66CDFA2</footer>`). The
[[notes/syllabus-doctrine|doctrine]] specifies a serial + register but the tool
(`pa-syllabus-stamp`) was never written. Systematize it into lectern.

## Scope (MVP)

`reg-syllabus stamp` + `reg-syllabus build` + register append, forward-only, on the
live Su26 repo. **Out of scope** (later cuts, doctrine already supports): the
back-catalog walker (`reg-syllabus walk` over the 34 GitHub repos) and
`reg-syllabus verify`. The serial function takes an optional `ref` now so the
walker is a thin addition later.

## The control number (serial)

A **repo-tree SHA-256** — NOT the HMAC `exam_serial`. `lectern/syllabus_serial.py`:

```python
def primary_md(repo: Path) -> Path          # README.md or syllabus.md (first found)
def compute_serial(repo: Path, ref: str | None = None) -> str
```

- File set: `git ls-files` (tracked only), **sorted by path**. With `ref`, read each
  blob at that commit (`git show <ref>:<path>`); else the working tree.
- For `primary_md` only: **strip the `serial:` and `revision-of:` frontmatter lines**
  before hashing (so the serial doesn't fingerprint itself).
- Hash: SHA-256 over, for each file in sorted order, `path.encode() + b"\0" + bytes`.
  Serial = `digest.hexdigest()[:8].upper()` (8 hex, uppercase — same family as
  `exam_serial`).
- Deterministic and content-addressed: identical tree ⇒ identical serial; any
  student-citable change ⇒ new serial.

## `reg-syllabus stamp <repo> [--revision-of XXXXXXXX] --vault-root V`

1. `serial = compute_serial(repo)` (working tree, serial-line stripped).
2. **Inject into `primary_md` frontmatter:** set `serial: <serial>`; if
   `--revision-of` given, set `revision-of: <prior>`. Create the `---` frontmatter
   block if absent. Idempotent — re-running on an unchanged tree is a no-op (same
   serial); on a changed tree, writes the new serial.
3. **Inject the markdown footer line** (for raw-source viewers), updating in place if
   present: `*Syllabus version <serial> · <YYYY-MM-DD>*`.
4. **Append a register row** to `<V>/notes/syllabus-serial-register.md` under a
   `## Live (Su26 forward)` section (created if absent): a table row
   `| <course> | <section> | <term> | <crn> | <serial> | <date> | <revision-of> |`.
   Parse `course/section/term/crn` from the repo dir name
   (`cecs-<course>-<term>-<section>-syllabus-<crn>`). Idempotent on `(serial)` —
   never double-append the same serial.

> Stamping is **manual** (run after a meaningful edit), per doctrine. The footer
> serial is *unexplained* (no instruction text references it) — present like a page
> number.

## `reg-syllabus build <repo> [--pdf]`

Render `primary_md` → distribution outputs, **serial read from frontmatter and
rendered in the footer**:

- **`syllabus.html`** — standalone styled HTML (pandoc + the shipped `syllabus.css`),
  footer `Serial · <serial>`. Default output.
- **`syllabus_canvas.html`** — Canvas-RCE-safe inline-styled fragment (generalizes
  `gen_canvas.py`: pandoc → HTML fragment, inject inline `style=` on structural
  elements, strip frontmatter/comments/anchor-nav/`<details>`). Default output.
- **`--pdf`** (opt-in only) → `syllabus.pdf` via pandoc + `eisvogel.latex`
  (print path — per [[feedback_pdfs_print_only_keys_markdown|PDFs-print-only]]).

Ship `eisvogel.latex` + `syllabus.css` as `lectern/references/` so every repo renders
consistently (today they're copied per-repo). The Canvas inline-style map
(ink/accent/muted/hair/panel + font stack) moves from `gen_canvas.py` into the module.

## Wrapper

`~/bin/reg-syllabus` → `python -m lectern.syllabus`, subcommands `stamp` / `build`
(mirrors `reg-qbank`).

## File structure

- **Create** `lectern/syllabus_serial.py` — `primary_md`, `compute_serial` (+ git helpers).
- **Create** `lectern/syllabus.py` — `stamp`, `build`, register-append, `main()` CLI.
- **Create** `lectern/references/syllabus.css`, `lectern/references/eisvogel.latex`
  (lifted from the Su26 repo).
- **Create** `tests/test_syllabus_serial.py`, `tests/test_syllabus.py`,
  `tests/fixtures/syllabus/` (a tiny git-repo fixture).
- **Create** `~/bin/reg-syllabus` wrapper.

## Testing

- **Serial determinism** — fixture repo → `compute_serial` stable across runs; changing
  a tracked file changes it; changing only the `serial:` line does NOT (strip works).
- **`ref` mode** — serial at first commit ≠ serial at HEAD after an edit.
- **stamp idempotency** — stamp twice on an unchanged tree → identical file + one
  register row; a changed tree → new serial + new row.
- **register parse** — repo dir name → `(course, section, term, crn)`.
- **build smoke** (`@needs_pandoc`) — produces `syllabus.html` + `syllabus_canvas.html`
  with the serial in the footer; `--pdf` produces `syllabus.pdf`.

## Risks

- **Pandoc/template availability** — build needs `pandoc` (+ a LaTeX engine for
  `--pdf`); gate those tests, fail the CLI with a clear message if absent.
- **Primary-md ambiguity** — a repo with both `README.md` and `syllabus.md`: prefer
  `syllabus.md`, else `README.md`; error if neither.
- **Serial self-reference** — the strip must run before hashing AND the footer line
  must be excluded from the hash (it lives in the rendered output, not the source md;
  the md footer line `*Syllabus version …*` IS hashed → acceptable, it's set by stamp
  and stable for a given serial... circular. **Resolution:** also strip the
  `*Syllabus version <hex> · <date>*` md footer line before hashing, same as the
  frontmatter serial lines).
